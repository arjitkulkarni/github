Common Branch Naming

  • feature/login-system
  • bugfix/error-500
  • hotfix/payment-bug
  • refactor/db-layer
  • release/1.0.0

Branch Workflow

  1. Create branch: git branch <branch-name>
  2. Switch: git checkout <branch-name> or git switch <branch-name>
  3. Create & switch: git checkout -b <branch-name>
  4. Make & commit changes
  5. Push branch: git push -u origin <branch-name>
  6. Merge to main, then delete branch when done

Branching Workflows

GitHub Flow: main ‚Üí feature branch ‚Üí PR ‚Üí review ‚Üí merge ‚Üí deploy

GitFlow: develop + feature/bugfix/hotfix/release (for larger projects)

Creating Branches

Branches allow you to work on features independently without affecting the main codebase. Use git branch or git checkout -b to create new branches.

Switching Between Branches

Use git checkout or git switch to move between branches. Your working directory updates to match the selected branch.

Branch Merging

Merge branches to integrate changes. Use git merge to combine branch history into your current branch, typically merging feature branches into main.

Branch Visualization

Visualize your branch structure with git log --oneline --graph --all. This shows how branches diverge and merge over time.

Branch Management

Delete merged branches with git branch -d to keep your repository clean. Use git branch -a to see all local and remote branches.
